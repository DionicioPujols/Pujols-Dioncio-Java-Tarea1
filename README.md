Descripcion:
Este repositorio contiene una aplicación de consola interactiva desarrollada en Java, diseñada como una tarea que integra la recopilacion y ejecucion los ejercicios prácticos asignados durante el estudio de los capítulos 1 al 5 del libro *Java 2: Manual de Programación*.

> Características de la tarea.

La aplicación centraliza todas las soluciones en un único punto de entrada mediante un "menú interactivo en consola". Este menú permite al usuario navegar de forma fluida entre 10 funcionalidades distintas y ejecutar cada ejercicio de manera iterativa hasta que decida finalizar el programa.

Funcionalidades Incluidas:

1. Calculadora Básica: Implementación de operaciones aritméticas elementales con captura de datos dinámicos.
2. Número Par o Impar: Control de flujos condicionales para evaluar propiedades numéricas.
3. Tabla de Multiplicar: Uso de estructuras cíclicas (for) para la generación de secuencias matemáticas.
4. Contador de Vocales: Manipulación de cadenas de texto (String) y análisis de caracteres.
5. Promedio de Notas: Procesamiento de colecciones numéricas para el cálculo de estadísticas básicas.
6. Clase Estudiante: Modelado de objetos con atributos específicos y métodos de comportamiento.
7. Cuenta Bancaria: Aplicación práctica de conceptos de Encapsulamiento mediante el uso de modificadores de acceso (private), constructores, métodos *getters* y *setters*.
8. Herencia de Vehículos: Demostración práctica de Herencia y Polimorfismos.
9. Matriz 3x3: Manipulación de arreglos bidimensionales, control de índices y recorrido de matrices por consola.
10. Sistema Básico de Inventario: Un mini sistema modular que gestiona productos en tiempo real, demostrando la interacción entre múltiples clases y colecciones dinámicas (ArrayList).

> Conceptos Técnicos Demostrados

Para lograr que el proyecto fuera limpio y escalable, el código se estructuró aplicando buenas prácticas de desarrollo:

* Modularidad: Todos los ejercicios y entidades del sistema (como en el Inventario o la Herencia de vehiculos) se ha desarrollado en su propia clase y archivo ".java" individual, evitando el código acoplado.
* Encapsulamiento Estricto: Protección de datos sensibles en las clases mediante atributos privados accesibles única y correctamente a través de métodos *getters* y *setters*.
* Estructuras de Control Robustas: Uso de bucles "do-while", selectores "switch-case" y condicionales para asegurar una experiencia de usuario fluida y libre de cierres inesperados.
